# Page Template Matrix - CUD Sustainability

## Purpose
This matrix defines repeatable page blueprint patterns for fast, consistent implementation across the sustainability architecture.

## Template Types

### T1 - Strategic Landing Page
Use for:
- Our Commitment
- Governance and Ethics
- Campus Operations (top level)

Blocks:
1. Hero with one key message and one CTA
2. Snapshot KPIs (3-6)
3. Pillar cards linking to child pages
4. Policy and framework strip
5. Evidence highlights
6. Related stories
7. Contact or collaboration CTA

Data fields:
- Intro summary
- Primary CTA label + link
- KPI label + value + reporting year
- Evidence links (URL, PDF)

### T2 - Program or Initiative Listing
Use for:
- Education
- Partnerships and Outreach
- News and Stories hub

Blocks:
1. Hero + filters
2. Featured item
3. Card grid list
4. Tag filters (SDG, category, year)
5. Pagination

Data fields:
- Item title
- Type
- SDG tags
- Date
- Outcome summary

### T3 - Evidence or Policy Detail Page
Use for:
- Policy pages
- Indicator evidence pages
- Governance minutes pages

Blocks:
1. Page summary
2. Policy metadata panel
3. Full text or embedded document
4. Compliance or indicator mapping
5. Revision history
6. Download and external source links

Data fields:
- Policy owner
- Approval date
- Last review date
- Next review date
- Related indicators

### T4 - Research Group or Center Profile
Use for:
- Center for Sustainability and Innovation
- Research groups and pillars

Blocks:
1. Group profile hero
2. Mission and scope
3. Members and roles
4. Active projects
5. Publications and outputs
6. SDG mapping
7. Collaboration CTA

Data fields:
- Group lead
- Focus areas
- Publications list
- Current projects
- Partner organizations

### T5 - Dashboard and Reporting Page
Use for:
- Reports and Disclosures
- Indicator Dashboard
- Submission Readiness Tracker

Blocks:
1. Reporting year selector
2. KPI tiles
3. Trend chart placeholders
4. Indicator table with status
5. Evidence links by indicator code
6. Download center

Data fields:
- Indicator code
- Category and lens
- Current status
- Source type
- Last updated date

## Minimum Content Requirements Before Publishing
1. Intro summary approved
2. At least one evidence source added
3. Last updated date visible
4. Owner/team contact specified
5. Related section links configured
